\doxysubsection{Detailed Description}
A node in the imported hierarchy.

Each node has name, a parent node (except for the root node), a transformation relative to its parent and possibly several child nodes. Simple file formats don\textquotesingle{}t support hierarchical structures -\/ for these formats the imported scene does consist of only a single root node without children.

\doxysubsubsection{\texorpdfstring{mName}{mName}}
{\footnotesize\ttfamily C\+\_\+\+STRUCT \mbox{\hyperlink{structai_string}{ai\+String}} ai\+Node\+::m\+Name}

The name of the node.

The name might be empty (length of zero) but all nodes which need to be referenced by either bones or animations are named. Multiple nodes may have the same name, except for nodes which are referenced by bones (see \mbox{\hyperlink{structai_bone}{ai\+Bone}} and \mbox{\hyperlink{structai_mesh_ad0388784814d4534ec011abf9b57e3a8}{ai\+Mesh\+::m\+Bones}}). Their names {\itshape must} be unique.

Cameras and lights reference a specific node by name -\/ if there are multiple nodes with this name, they are assigned to each of them. ~\newline
 There are no limitations with regard to the characters contained in the name string as it is usually taken directly from the source file.

Implementations should be able to handle tokens such as whitespace, tabs, line feeds, quotation marks, ampersands etc.
