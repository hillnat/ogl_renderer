\doxysubsection{Detailed Description}
Helper structure to describe a virtual camera.

Cameras have a representation in the node graph and can be animated. An important aspect is that the camera itself is also part of the scene-\/graph. This means, any values such as the look-\/at vector are not {\itshape absolute}, they\textquotesingle{}re {\bfseries{relative}} to the coordinate system defined by the node which corresponds to the camera. This allows for camera animations. For static cameras parameters like the \textquotesingle{}look-\/at\textquotesingle{} or \textquotesingle{}up\textquotesingle{} vectors are usually specified directly in \mbox{\hyperlink{structai_camera}{ai\+Camera}}, but beware, they could also be encoded in the node transformation. The following (pseudo)code sample shows how to do it\+: ~\newline
~\newline
 
\begin{DoxyCode}{0}
\DoxyCodeLine{\textcolor{comment}{// Get the camera matrix for a camera at a specific time}}
\DoxyCodeLine{\textcolor{comment}{// if the node hierarchy for the camera does not contain}}
\DoxyCodeLine{\textcolor{comment}{// at least one animated node this is a static computation}}
\DoxyCodeLine{get-\/camera-\/matrix (node sceneRoot, camera cam) : matrix}
\DoxyCodeLine{\{}
\DoxyCodeLine{   node   cnd = find-\/node-\/\textcolor{keywordflow}{for}-\/camera(cam)}
\DoxyCodeLine{   matrix cmt = \mbox{\hyperlink{group__ext__matrix__transform_ga81696f2b8d1db02ea1aff8da8f269314}{identity}}()}
\DoxyCodeLine{}
\DoxyCodeLine{   \textcolor{comment}{// as usual -\/ get the absolute camera transformation for this frame}}
\DoxyCodeLine{   for each node nd in hierarchy from sceneRoot to cnd}
\DoxyCodeLine{     matrix cur}
\DoxyCodeLine{     if (is-\/animated(nd))}
\DoxyCodeLine{        cur = eval-\/animation(nd)}
\DoxyCodeLine{     else cur = nd-\/>mTransformation;}
\DoxyCodeLine{     cmt = mult-\/matrices( cmt, cur )}
\DoxyCodeLine{   end for}
\DoxyCodeLine{}
\DoxyCodeLine{   \textcolor{comment}{// now multiply with the camera's own local transform}}
\DoxyCodeLine{   cam = mult-\/matrices (cam, get-\/camera-\/matrix(cmt) )}
\DoxyCodeLine{\}}

\end{DoxyCode}


\begin{DoxyNote}{Note}
some file formats (such as 3DS, ASE) export a \char`\"{}target point\char`\"{} -\/ the point the camera is looking at (it can even be animated). \mbox{\hyperlink{namespace_assimp}{Assimp}} writes the target point as a subnode of the camera\textquotesingle{}s main node, called \char`\"{}$<$cam\+Name$>$.\+Target\char`\"{}. However this is just additional information then the transformation tracks of the camera main node make the camera already look in the right direction. 
\end{DoxyNote}
