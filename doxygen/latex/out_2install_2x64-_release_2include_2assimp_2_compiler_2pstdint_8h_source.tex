\DoxyCodeLine{41 \textcolor{comment}{ *  applications including cryptography, hashing, variable length}}
\DoxyCodeLine{42 \textcolor{comment}{ *  integer libraries and so on.  But for most developers its likely}}
\DoxyCodeLine{43 \textcolor{comment}{ *  useful just for programming sanity.}}
\DoxyCodeLine{44 \textcolor{comment}{ *}}
\DoxyCodeLine{45 \textcolor{comment}{ *  The problem is that some compiler vendors chose to ignore the C99}}
\DoxyCodeLine{46 \textcolor{comment}{ *  standard and some older compilers have no opportunity to be updated.}}
\DoxyCodeLine{47 \textcolor{comment}{ *  Because of this situation, simply including stdint.h in your code}}
\DoxyCodeLine{48 \textcolor{comment}{ *  makes it unportable.}}
\DoxyCodeLine{49 \textcolor{comment}{ *}}
\DoxyCodeLine{50 \textcolor{comment}{ *  So that's what this file is all about.  Its an attempt to build a}}
\DoxyCodeLine{51 \textcolor{comment}{ *  single universal include file that works on as many platforms as}}
\DoxyCodeLine{52 \textcolor{comment}{ *  possible to deliver what stdint.h is supposed to.  Even compilers}}
\DoxyCodeLine{53 \textcolor{comment}{ *  that already come with stdint.h can use this file instead without}}
\DoxyCodeLine{54 \textcolor{comment}{ *  any loss of functionality.  A few things that should be noted about}}
\DoxyCodeLine{55 \textcolor{comment}{ *  this file:}}

\DoxyCodeLine{58 \textcolor{comment}{ *       interface on all platforms.  The extreme variability of the}}
\DoxyCodeLine{59 \textcolor{comment}{ *       ANSI C standard makes this an impossibility right from the}}
\DoxyCodeLine{60 \textcolor{comment}{ *       very get go. Its really only meant to be useful for the vast}}
\DoxyCodeLine{61 \textcolor{comment}{ *       majority of platforms that possess the capability of}}
\DoxyCodeLine{62 \textcolor{comment}{ *       implementing usefully and precisely defined, standard sized}}
\DoxyCodeLine{63 \textcolor{comment}{ *       integer scalars.  Systems which are not intrinsically 2s}}
\DoxyCodeLine{64 \textcolor{comment}{ *       complement may produce invalid constants.}}
\DoxyCodeLine{65 \textcolor{comment}{ *}}
\DoxyCodeLine{66 \textcolor{comment}{ *    2) There is an unavoidable use of non-\/reserved symbols.}}
